## 自选股实时行情查询逻辑调整说明

### 变更背景
- 原有 `watchlist_manage.py` 在缺失行情时会调用 AkShare 多个实时接口补齐数据，并回写到 `stock_realtime_quote`。
- 线上环境希望只依赖内部采集的实时行情表，避免频繁向外部接口发起调用造成不稳定或限频风险。

### 实施要点
- 移除 AkShare 依赖与补数函数，保留 `safe_float` 等基础工具函数。
- 查询逻辑仅访问 `stock_realtime_quote`：
  - 优先读取当日记录（通过 `trade_date like '{today}%'`）。
  - 若当日无数据，则自动回退至 `trade_date` 最近的一天。
  - 无论是否命中行情，都会返回自选股基础信息，行情字段允许为空。
- 相关导入精简：删除 `akshare` 及 `timedelta` 等不再使用的模块。

### 校验步骤
1. 确保采集任务已同步实时行情到 `stock_realtime_quote`（至少包含最近交易日数据）。
2. 调用接口 `GET /api/watchlist`：
   - 日内请求应返回当日行情，`update_time` 由表内数据决定。
   - 若手动清空当日记录，再次请求应自动回退至最新的历史交易日。
   - 网络断开、AkShare 不可用时仍应成功返回数据（仅依赖本地库）。
3. 前端 watchlist 页面验证：
   - 刷新页面不再触发外部实时调用，响应速度保持稳定。
   - 对于缺失行情的股票，数值显示为空或 `-`，但卡片仍显示分组/名称。

### 注意事项
- 该改动假设外部采集进程负责更新 `stock_realtime_quote`，若存在延迟请新增相应监控。
